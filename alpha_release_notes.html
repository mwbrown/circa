Alpha Known Issues

Here are some of the major known issues in Circa's alpha. All of these are planned to be fixed/improved in future releases.

Performance problems

There are a few areas where runtime performance is currently not good enough. One specific area is with stateful values. Having a loop like this:

for i in range(1000)
  state i
  ...
end

might your FPS to drop.

No recursion

Circa-defined functions don't support reentrancy and can't be called recursively. This will be fixed in a future release.

Not enough 3rd party libraries

In alpha 1, we only have bindings for drawing images, rendering TTF fonts, and doing a few OpenGL commands. In a future release we'd like to include libraries for physics, audio, more kinds of 2d rendering, more kinds of 3d rendering, and other things.

Unstable syntax and API

The syntax and API of Circa may change in a future version. Code that works against alpha-1 is not guaranteed to work in future releases.

Return doesn't return

The 'return' keyword is used to pass back a value from a function to the caller, but in other C-like languages it's also used for control flow. Circa doesn't currently support using return for control flow, which can lead to surprising results. The following function:

def my_func() : int
  if true
    return 1
  end
  return 2
end

will always return 2, even though it seems like it should return 1. This will be fixed.

No parametrized types

Circa tries to be a statically typed language, but it isn't 100% there yet. One problem is that there's no support for parametrized types, like a List<Something>, like you might expect. The most common way this problem manifests is that you need to manually cast the iterator in a for loop, using the : operator.

-- things is a list of Thing items
for thing in things
  -- this line is necessary to use 'thing' as a Thing:
  thing = thing : Thing
  ...
end
